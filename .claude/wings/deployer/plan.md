# wing: deployer (WIP)

## цель
Telegram бот для развёртывания VPN-кластера на VPS пользователя.
Без знаний — пользователь вводит SSH данные и токен бота, получает готовый кластер.

## сценарии (выбор при старте)
1. **direct** — 1 зарубежный сервер, Reality по IP, подписка с одним сервером
2. **cascade** — RU entry + FI/SE exit, серверная маршрутизация (по типу goida.fun)
   - опция: добавить "home" сервер для обхода белых списков (inbound-10004 аналог)

## архитектура деплой-бота
- Телеграм бот (Python async, aiogram или telebot)
- Асинхронная SSH-песочница на пользователя (asyncssh или paramiko в thread)
- Состояние сессии: Redis или SQLite
- Интервью → план → деплой → верификация → выдача подписки

## интервью (вопросы к пользователю)
1. Сценарий: direct / cascade?
2. SSH: host, port, user, key/password
3. Токен Telegram бота (для управления после деплоя)
4. Домен или IP? (для Reality и сертификата)
5. [cascade] Адреса FI/SE серверов
6. Количество пользователей (для настройки лимитов)

## деплой-скрипт (план)
### direct
- установка Xray-core
- генерация Reality ключей
- настройка одного VLESS+Reality inbound
- генерация подписки → отдаём пользователю

### cascade
- RU нода: 3X-UI + Xray + zapret/nfqws + AdGuard Home
- FI/SE ноды: минимальный Xray + Reality inbound
- routing rules по runetfreedom geo-файлам
- Observatory + leastPing балансер
- vpn-bot.py (упрощённая версия без smart-pro)
- sub-updater (опционально)

## безопасность
- SSH ключ/пароль не хранится после деплоя
- Все секреты генерируются на стороне сервера пользователя
- Наш бот только инициирует SSH сессию, не хранит данные
