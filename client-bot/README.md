# goida client bot

Отдельный Telegram-бот и HTTP API для клиентской Mini App.

## что делает

- отдаёт статическую Mini App из `client-web/`;
- читает пользователей и устройства из основного `bot.db`;
- хранит клиентские настройки тарифа/оплаты в отдельных таблицах;
- даёт админу ручные команды для срока оплаты, лимита устройств и рассылок;
- напоминает об оплате 14 числа каждого месяца.

## запуск

```bash
cp client-bot/.env.example client-bot/.env
# заполнить CLIENT_BOT_TOKEN и CLIENT_WEBAPP_URL
python3 client-bot/client-bot.py
```

## команды админа

- `/broadcast текст` — рассылка всем привязанным клиентам.
- `/setpaid username YYYY-MM-DD` — указать, до какой даты оплачено.
- `/paid username N` — прибавить N месяцев от сегодня/текущего срока.
- `/setdevices username N` — изменить лимит устройств.
- `/who username` — карточка клиента.
- `/remind` — вручную отправить напоминания тем, у кого оплата сегодня.

## команды клиента

- `/start` — открыть Mini App.
- `/link username` — привязать Telegram к пользователю из `bot.db`, если `tg_id` ещё не заполнен в основной базе.
