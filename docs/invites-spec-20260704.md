# Инвайты — спека (2026-07-04)

## Цель

Рефералка «для своих»: действующие пользователи приглашают друзей, получают за это бесплатные дни.
Цель проекта — не заработать, а окупать инфру и давать людям доступ в интернет; маржинальная
стоимость юзера ~0, поэтому валюта наград — дни подписки.

## Экономика

- **Друг**: 7 дней бесплатного триала. Дальше без скидок — либо оплачивает тариф, либо отключаем.
- **Инвайтер**: разовый бонус **+30 дней** за **первую оплату** приглашённого друга.
  - Начисляется автоматически в момент, когда владелец подтверждает первый перевод друга.
  - Сдвиг `expireAt` инвайтера; если подписка истекла — 30 дней от «сейчас».
  - Инвайтеру уходит нотификация «твой друг оплатил, +30 дней».
- **Кто может приглашать**: только юзеры с хотя бы одной оплатой в истории. Триальщики — нет
  (отсекает цепочки триал→триал→триал).
- ⚠️ **Логика начисления должна быть заменяемой**: единая функция вида
  `reward_days(inviter_profile, friend_profile) -> int` (сейчас константа 30) + запись применённого
  значения в данные инвайта. Владелец хочет потом пересмотреть честность: тарифы разные
  (2 устройства — 170₽, 7 устройств — 320₽), а бонус одинаковый.

## Два типа инвайтов

### 1. Постоянная реф-ссылка (для друзей с работающим Telegram)

- У каждого имеющего право приглашать юзера — постоянный `ref_code`,
  ссылка `t.me/<client-bot>?start=ref_<code>`.
- Переход → авто-создание юзера → триал 7 дней → обычный флоу client-bot / Mini App.
- Инвайтер фиксируется по **first-touch** в момент активации.

### 2. Личное приглашение (для друзей без Telegram / с неработающим Telegram)

Решает курицу-и-яйцо: без VPN не работает Telegram, без Telegram не получить VPN.

- Одноразовый токен, выдаёт **сразу ссылку на подключение** (subscription URL).
- Движок подписки собирает такому юзеру профиль из двух записей:
  1. `Оптимальный 🇸🇨 [1 час]` — настоящий инбаунд, smart-роутинг, лимит 1 устройство;
  2. `❗️ПРОЧИТАЙТЕ ОПИСАНИЕ❗️` — фейк-запись; в описании: доступ временный, нажми самолётик
     и привяжи Telegram; внутри персональный deep-link `t.me/<client-bot>?start=link_<token>`.
- **Отсчёт часа — от первого фетча подписки** (`first_fetch_at`), не от создания инвайта.
- Привязал TG в течение часа → полноценный триал 7 дней, инбаунды заменяются на нормальные.
- **Не привязал за час** → движок отдаёт фейк-инбаунды с текстом
  «доступ истёк — привяжи телеграм, чтобы продолжить»; **deep-link остаётся рабочим ещё 7 дней**
  (grace: человек мог отвлечься; VPN не работает, но самоспасение возможно).
- Токен привязки умирает через 7 дней после `first_fetch_at` → статус `expired`.
- Безопасность: инвайт-токен одноразовый, гасится при активации; до активации страница/подписка
  не светит ничего лишнего; длинные случайные токены, отзыв инвайта из Mini App до активации.

## Модерация (гибрид)

- Триал стартует автоматически, одобрение владельца нужно только для перехода в платные.
- При активации триала владельцу в админ-бот прилетает нотификация:
  - текст: «у X появился друг Y», **обязательно tg_id и @username** друга;
  - кнопки: **Одобрить** (style success) / **Забанить** (style danger);
  - Одобрить → у друга открывается возможность оплаты (кнопка в Mini App / client-bot);
  - Забанить → выдача подписки заменяется на фейк-инбаунды;
  - ничего не нажато → триал живёт, оплата закрыта до одобрения.

## Дедуп и self-invite

- Существующий юзер (по tg_id) перешёл по чужой реф-ссылке → ничего не создаём, инвайт не
  засчитываем; вежливо сообщаем: «ты уже был приглашён Петей» (называем первого инвайтера).
- Открыл собственную ссылку → «это твоя ссылка».
- Два инвайта на одного человека → побеждает первая активация.
- Self-invite (свой tg_id по своей ссылке) — невозможен по определению выше.

## Анти-абьюз

- Лимитов на количество приглашений **нет** — на доверии (приглашают только платящие).
- Вместо лимитов — **полный структурированный аудит**: логгируем каждое действие с каждой
  пригласительной ссылкой (создание, открытие, фетч подписки, активация, привязка TG, отзыв,
  истечение, одобрение/бан, начисление бонуса) с timestamp, ip/UA где применимо.
  Отдельная таблица событий, чтобы подозрительную активность было видно и можно было разбирать.

## Данные

Таблица `invites`:

| поле | описание |
|---|---|
| id | pk |
| type | `ref` / `personal` |
| inviter_username | Remnawave username инвайтера |
| token | инвайт-токен (для `ref` — постоянный `ref_code`) |
| created_at | создание |
| first_fetch_at | первый фетч подписки (старт часа, только `personal`) |
| tg_linked_at | момент привязки Telegram |
| activated_username | созданный друг |
| activated_tg_id | tg_id друга |
| status | `created → opened → tg_pending → trial → approved → paid / expired / revoked / banned` |
| reward_days | применённое значение бонуса |
| reward_applied_at | момент начисления |

Таблица `invite_events`: invite_id, event, ts, meta (json: ip, ua, actor).

## UI — раздел «инвайты» в Mini App

- Своя постоянная реф-ссылка + QR + готовый текст для пересылки (WhatsApp/iMessage/лично).
- Кнопка «создать личное приглашение» (для друга без TG) → ссылка + QR + готовый текст.
- Список приглашённых со статусами (создан → открыт → триал → ждёт одобрения → оплатил ✓).
- Счётчик заработанных дней.
- Раздел активен только для имеющих право приглашать; остальным — «пригласи после первой оплаты».

## Точки имплементации

- **client-bot** (`client-bot/client-bot.py`): deep-links `ref_<code>` / `link_<token>`,
  создание триал-юзеров, привязка TG, сообщения дедупа, нотификации инвайтеру.
- **subscription engine** (`subscription/engine.py`): режимы выдачи — временный час
  (реальный инбаунд + фейк-инструкция), истёкший час (фейк «привяжи телеграм»),
  бан (фейк «подписка закончилась»), нормальный триал/платный.
- **vpn-bot (админ)** (`bot/vpn-bot.py`): нотификация о новом друге с кнопками Одобрить/Забанить,
  хук начисления бонуса при подтверждении первой оплаты друга.
- **client-web** (`client-web/index.html`): заменить заглушку `panelInvites()` на живой раздел.
